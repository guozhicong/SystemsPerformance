## 基础介绍

### C2 优化

C2 编译器识别到热点方法后，基于当前执行数据做激进预判（如预判某分支永远执行），删除无用逻辑并将字节码编译为高度优化的机器码，异步替换解释执行以极致提升性能。

### C2 退优化

当 C2 的激进预判失效（如预判的分支未执行），JVM 会同步将执行方式从机器码切回解释执行，标记失效机器码为不可进入，重置调用计数器，未达重编译阈值则后续可重新编译，否则永久解释执行（永久的解释执行可能带来cpu冲高问题，且无法恢复）。

```java
/**
 * JDK8关闭分层编译时，计算密集型代码触发C2反优化示例
 * 修复点：
 * 1. 补充Deopt前后的耗时对比
 * 2. 修正耗时统计逻辑
 * 3. 清晰区分C2编译/Deopt/重编译的耗时差异
 */
public class C2DeoptCalcExample {

    // 核心计算方法：根据数值奇偶性做不同累加逻辑（易触发分支预测优化）
    public static long calcSum(int[] nums) {
        long sum = 0;
        // 热点计算循环：C2会预判分支走向（比如全是偶数）并编译机器码
        for (int num : nums) {
            if (num % 2 == 0) { // 偶数分支：简单累加
                sum += num * 2;
            } else { // 奇数分支：复杂计算（乘法+取模）
                sum += (num * 3 + 7) % 5;
            }
        }
        return sum;
    }

    public static void main(String[] args) throws InterruptedException {
        // ========== 初始化数据 ==========
        // 构造全偶数数组（触发C2编译）
        int[] evenNums = new int[100000];
        for (int i = 0; i < evenNums.length; i++) {
            evenNums[i] = i * 2; // 0,2,4,6... 全偶数
        }

        // 构造全奇数数组（触发Deopt）
        int[] oddNums = new int[100000];
        for (int i = 0; i < oddNums.length; i++) {
            oddNums[i] = i * 2 + 1; // 1,3,5,7... 全奇数
        }

        // ========== 阶段1：高频执行偶数计算，触发C2编译 ==========
        long startTime, endTime, costTime;
        System.out.println("===== 阶段1：高频执行偶数计算，触发C2编译 =====");
        startTime = System.currentTimeMillis();
        // 高频调用（5万次），触发C2编译（JDK8默认1万次触发）
        for (int i = 0; i < 50000; i++) {
            calcSum(evenNums);
        }
        endTime = System.currentTimeMillis();
        costTime = endTime - startTime;
        System.out.println("C2编译完成 - 5万次偶数计算总耗时：" + costTime + " 毫秒");
        System.out.println("单次偶数计算平均耗时：" + (costTime / 50000.0) + " 毫秒");
        Thread.sleep(1000); // 等待C2编译线程完成

        // ========== 阶段2：执行奇数计算，触发Deopt ==========
        System.out.println("\n===== 阶段2：执行奇数计算，触发C2反优化 =====");
        // 先执行1次奇数计算（触发Deopt）
        startTime = System.currentTimeMillis();
        long oddResult = calcSum(oddNums);
        endTime = System.currentTimeMillis();
        costTime = endTime - startTime;
        System.out.println("Deopt后 - 1次奇数计算耗时：" + costTime + " 毫秒");
        System.out.println("奇数计算结果：" + oddResult);
        Thread.sleep(2000); // 等待Deopt完成

        // ========== 阶段3：验证Deopt后偶数计算的性能退化 ==========
        System.out.println("\n===== 阶段3：Deopt后再次执行偶数计算（验证性能退化） =====");
        startTime = System.currentTimeMillis();
        for (int i = 0; i < 10100; i++) {
            calcSum(evenNums);
        }
        endTime = System.currentTimeMillis();
        costTime = endTime - startTime;
        System.out.println("Deopt后 - 1万次偶数计算耗时：" + costTime + " 毫秒");
        System.out.println("Deopt后单次偶数计算平均耗时：" + (costTime / 10000.0) + " 毫秒");
        Thread.sleep(1000); // 等待C2编译线程完成
        // ========== 阶段4：验证重编译后的性能恢复 ==========
        System.out.println("\n===== 阶段4：高频执行触发重编译，验证性能恢复 =====");
        startTime = System.currentTimeMillis();
        // 再次高频调用（1万次），触发C2重编译
        for (int i = 0; i < 10000; i++) {
            calcSum(evenNums);
        }
        endTime = System.currentTimeMillis();
        costTime = endTime - startTime;
        System.out.println("重编译后 - 1万次偶数计算耗时：" + costTime + " 毫秒");
        System.out.println("重编译后单次偶数计算平均耗时：" + (costTime / 10000.0) + " 毫秒");
	Thread.sleep(1000); // 等待C2编译线程完成
    }
}
```

## 日志分析

1. 带%表示触发了C2编译

   277    3 %           C2DeoptCalcExample::calcSum @ 11 (63 bytes)

2. made not entrant ： 当 C2 编译的某个方法机器码因反优化（比如分支预判失败）失效后，JVM 会把这个版本的机器码标记为`made not entrant`——**后续所有新的调用都不能再进入这个机器码执行，必须走解释执行或等待重编译**。



## JVM 参数

查看JVM默认的启动参数 `java -XX:+PrintFlagsFinal -version`

| JVM参数                             | 描述                                                         | 备注                              |
| ----------------------------------- | ------------------------------------------------------------ | --------------------------------- |
| -XX:ReservedCodeCacheSize           | JIT 编译后的代码都会放在 CodeCache 里。如果这个空间不足，JIT 就无法继续编译，编译执行会变成解释执行，性能会降低一个数量级。 | 怎么查看剩余的CodeCache空间大小？ |
| -XX:-TieredCompilation              | 关闭分层编译优化。关闭后，JVM 会只使用单一编译器（默认是 C2，除非配合`TieredStopAtLevel=1`指定 C1）。 |                                   |
| -XX:PerMethodRecompilationCutoff=10 | 当你的`calcSum`方法因 Deopt 被标记为`made not entrant`后，JVM 会尝试重编译它；但最多只能重编译 10 次，第 11 次 Deopt 后，JVM 会**永久放弃 C2 编译**，该方法后续只能以解释执行运行。 | 默认400                           |
| -XX:CompileThreshold=10000          | 控制 C2 编译器触发阈值的核心参数                             | 默认10000                         |
| -XX:CompileOnly=none                | 将所有的编译优化关闭，仅做解释执行                           |                                   |
| PerBytecodeRecompilationCutoff      |                                                              | 默认200                           |
